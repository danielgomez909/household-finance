Overview

A single-page HTML app for household financial management. It pulls spending and budget data from YNAB exports, tracks assets and liabilities across all accounts, models envelope-based budget plan vs actual performance, and supports what-if income/spending scenario modeling.

Primary use case at build time: Modeling the cash flow impact of a potential job change with lower initial income — showing how spending adjustments affect monthly surplus/deficit and goal timelines, and what YNAB envelope targets would need to change.

Usage cadence: Monthly for full review; certain views (cash flow, plan accuracy) checked weekly.

Plan Tab — Detail

Envelope volatility

Shows how much each envelope's assigned amount varies month to month. Flags envelopes exceeding a user-set threshold. Toggle between % (coefficient of variation) and $ (std dev) mode. Slider adjusts threshold. Groups are collapsible. Group-level totals and variability shown in header row.

Period selector: YTD / 6 / 12 months (past months only).

Envelope comparison

Assigned vs actual per envelope per month with target as reference. Cells flagged red when actual exceeds target/assigned by threshold; green when 10%+ under. Toggle % or $ threshold, slider range $50–$5,000. Category group totals row. Groups collapsible.

Period selector: YTD / 3 / 6 / 12 months.

Forward plan

Unified table: historical months show Assigned / Actual / Target; forecast months show Assigned / Target. Historical months are lightly shaded. Separate period selectors for Historical (None / YTD / 3 / 6 / 12 mos) and Forecast (None / EOY / 3 / 6 / 12 mos). Total and Avg/mo summary columns. Groups collapsible.

Target lookup: getTarget() matches across all envelope_targets rows (active and inactive) by date range, so historical inactive rows correctly supply targets for the months they covered.

Admin Tab — Detail

Plan import

YNAB's Export function produces two files. Upload both here:

  • Register CSV — transaction-level actuals. Aggregated by month/category. Slalom payroll detected by payee name → payee_tag='slalom'; all other inflows → 'other'. Replaces all previous ynab_import transactions. Skips Hidden Categories.
  • Plan CSV — monthly assigned amounts per envelope. Stored in plans. Replaces previous plan data. Skips Hidden Categories and Credit Card Payments. Data imported from Jan 2025 forward.

File inputs reset on click so you can re-select the same file without refreshing the page. Each upload clears and replaces — safe to re-run.

Envelope targets

Manually set monthly targets for each envelope. Category and envelope dropdowns populate from plan import data. Selecting an envelope shows all existing rows (active + inactive) so you can check for overlaps before saving.

What-If Tab — Design Notes

Computes a monthly average baseline from the selected period and compares to a user-defined scenario:

  • Baseline: Average monthly income + average spending per category group for selected period (YTD / 3 / 6 / 12 months)
  • Scenario: New monthly income + optional per-group spending adjustments in $ or %
  • Output: Side-by-side monthly surplus/deficit, monthly and annual delta, goal/envelope timeline impact
  • 3-paycheck detection: Income months exceeding period average by >30% are flagged
  • Non-Slalom income: Months with payee_tag='other' inflow rows flagged with ★

Known Limitations

  • Envelope target date ranges not auto-enforced for contiguity — user responsibility
  • YNAB Plan CSV month format assumed to be "Mon YYYY" (e.g. "Jan 2026") — different locales may fail
  • No authentication — app is publicly accessible at the Vercel URL
  • Supabase anon key embedded in HTML — fine for personal use, not for shared apps
  • Category groups are hardcoded to YNAB structure — changes to YNAB group names require code updates
